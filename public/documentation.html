Authentication

Most endpoints require authentication using a JSON Web Token (JWT). To obtain a token, you must first register a user (if you haven't already) and then log in via the /login endpoint.

Once you have a token, include it in the Authorization header for all protected requests:

Authorization: Bearer [YOUR_JWT_TOKEN]

Important: The JWT signing and verification process relies on a secret key (JWT_SECRET) configured on the server using environment variables.

POST /login

Authenticates a user and returns user data along with a JWT.

Authentication: Public

Request Body:
  • username (Required, String)
  • password (Required, String)
{ "username": "testuser", "password": "password123" }
Success Response (200 OK):

Returns an object containing user details (excluding password) and the JWT.

{ "user": { "_id": "...", "username": "...", ... }, "token": "eyJ..." }
Error Responses: 401 (invalid credentials), 500

User Endpoints

POST /users

Registers a new user account.

Authentication: Public

Request Body:
  • username (Required, String): Min 5 alphanumeric.
  • password (Required, String): Min 8 chars.
  • email (Required, String): Valid email format.
  • birthday (Optional, String): "YYYY-MM-DD".
  • firstname (Optional, String)
  • lastname (Optional, String)
{ "username": "newuser", "password": "password123", "email": "new@example.com", ... }
Success Response (201 Created): Returns new user object (no password). 
{ "_id": "...", "username": "newuser", ... }
Error Responses: 400 (validation, duplicate), 500

GET /users/:username

Retrieves a user's profile information by username.

Authentication: JWT Required (User must match :username).

URL Parameters: :username (Required) Success Response (200 OK): Returns user object (no password) with populated favorites. 
{ "_id": "...", "username": "...", "favoriteMovies": [ { "_id": "...", "title": "..." } ] }
Error Responses: 401, 403, 404, 500

PUT /users/:username

Updates a user's profile information.

Authentication: JWT Required (User must match :username).

URL Parameters: :username (Required) Request Body: (Optional fields: username, email, birthday, firstname, lastname) 
{ "email": "updated@example.com" }
Success Response (200 OK): Returns updated user object (no password). 
{ "_id": "...", "username": "...", ... }
Error Responses: 400 (validation, duplicate, no fields), 401, 403, 404, 500

POST /users/:username/favorites/:movieId

Adds a movie to the user's list of favorite movies.

Authentication: JWT Required (User must match :username).

URL Parameters: :username (Required), :movieId (Required, ObjectId) Success Response (200 OK): Returns updated user object with populated favorites. 
{ "_id": "...", "username": "...", "favoriteMovies": [ ... ] }
Error Responses: 400 (Invalid ID), 401, 403, 404 (User or Movie), 500

DELETE /users/:username/favorites/:movieId

Removes a movie from the user's list of favorite movies.

Authentication: JWT Required (User must match :username).

URL Parameters: :username (Required), :movieId (Required, ObjectId) Success Response (200 OK): Returns updated user object with populated favorites. 
{ "_id": "...", "username": "...", "favoriteMovies": [ ... ] }
Error Responses: 400 (Invalid ID), 401, 403, 404 (User), 500

DELETE /users/:username

Deregisters a user account.

Authentication: JWT Required (User must match :username).

URL Parameters: :username (Required) Success Response (200 OK): 
{ "message": "User account \"...\" deleted successfully." }
Error Responses: 401, 403, 404, 500

  • genre(Req, Ref)
  • director(Req, Ref)
  • actors(Array[Ref])
  • imagePath
  • featured
  • releaseYear
  • rating

  • _id
  • username(Req, Unique)
  • password(Req, Hash)
  • email(Req, Unique)
  • birthday
  • firstname
  • lastname
  • favoriteMovies(Array[Ref])

  • _id
  • name(Req, Unique)
  • description(Req)

Director Model Schema

  • _id
  • name(Req, Unique)

  • birth(Req)

Actor Model Schema

  • bio(Req)
