Movie API Documentation

Welcome to the documentation for MovieMobs API. This API provides access to information about movies, actors, directors, and genres.

API Endpoints

Endpoint Method Description
Authentication Required for All Endpoints Below (except POST /users)
A valid JWT token must be included in the 'Authorization' header as a Bearer token.
/ GET Provides a welcome message.
/movies GET Retrieves a list of all movies available.
/movies/title/:title GET Finds and returns information about a single movie, identified by its title.
/movies/genre/:genreName GET Retrieves a list of movies that belong to a specific genre.
/movies/director/:directorName GET Retrieves a list of movies directed by a particular director.
/movies/:movieId GET Finds and returns information about a single movie, identified by its unique ID.
/movies POST Creates a new movie entry. Requires providing movie details in a specific format.
/movies/:movieId PUT Updates information for an existing movie, using its unique ID. Requires providing the updated movie details.
/movies/:movieId DELETE Removes a movie from the database, using its unique ID.
/movies/:movieId/actors GET Retrieves a list of actors who starred in a specific movie.
/users POST Registers a new user account. Requires providing user details in a specific format.
/users/:username PUT Updates information for an existing user account, using their username. Requires providing the updated user details.
Authorization: User can only update their own profile.
/users/:username/favorites/:movieId POST Adds a movie to a user's list of favorite movies.
/users/:username/favorites/:movieId DELETE Removes a movie from a user's list of favorite movies.
/users/:username DELETE Deletes a user account.
/genres GET Retrieves a list of all genres.
/genres/:name GET Finds and returns information about a single genre, identified by its name.
/genres POST Creates a new genre entry. Requires providing genre details in a specific format.
/genres/:name PUT Updates the description of an existing genre, using its name. Requires providing the updated description.
/genres/:name DELETE Removes a genre from the database, using its name.
/directors GET Retrieves a list of all directors.
/directors/name/:name GET Finds and returns information about a single director, identified by their name.
/directors/:directorId GET Finds and returns information about a single director, identified by their unique ID.
/directors POST Creates a new director entry. Requires providing director details in a specific format.
/directors/:directorId PUT Updates information for an existing director, using their unique ID. Requires providing the updated director details.
/directors/:directorId DELETE Removes a director from the database, using their unique ID.
/actors GET Retrieves a list of all actors.
/actors/name/:name GET Finds and returns information about a single actor, identified by their name.
/actors/:actorId GET Finds and returns information about a single actor, identified by their unique ID.
/actors POST Creates a new actor entry. Requires providing actor details in a specific format.
/actors/:actorId PUT Updates information for an existing actor, using their unique ID. Requires providing the updated actor details.
/actors/:actorId DELETE Removes an actor from the database, using their unique ID.
/admin/users GET (Admin) Retrieves a list of all user accounts.
/admin/movies GET (Admin) Retrieves a list of all movies with detailed information.
/admin/genres GET (Admin) Retrieves a list of all genres.
/admin/directors GET (Admin) Retrieves a list of all directors.
/admin/actors GET (Admin) Retrieves a list of all actors.

Endpoint Details

Get all movies

GET /movies

This endpoint is like asking a librarian for a list of all the movies in the library. It gives you back a collection of all the movie entries stored in the database. Each movie entry is like a card with details such as title, description, genre, director, and actors. 

                    [
                        {
                            "_id": "string (ObjectId)",
                            "title": "string (required)",
                            "description": "string (required)",
                            "genre": { "name": "string", "description": "string" },
                            "director": { "name": "string", "bio": "string", "birth": "date (ISO 8601, e.g., 'YYYY-MM-DD')", "death": "date (ISO 8601, optional)" },
                            "actors": ["string (Actor Name)"],
                            "imagePath": "string (optional - URL to movie poster)",
                            "featured": "boolean (optional - indicates if movie is featured)",
                            "__v": "number"
                        },
                        // ... more movies
                    ]

Status Codes: 200 OK (Here's the list!), 500 Internal Server Error (Oops, something went wrong on our end!)

Get movie by title

GET /movies/title/:title

This endpoint is like asking the librarian for a specific movie by its title. You provide the title, and it gives you back the detailed information about that one movie. If there's no movie with that title, it will tell you. 

                    { "_id": "string (ObjectId)", "title": "string (required)", "description": "string (required)", "genre": { ... }, "director": { ... }, "actors": ["string (Actor Name)"], "imagePath": "string (optional - URL to movie poster)", "featured": "boolean (optional - indicates if movie is featured)", "__v": "number" }

Status Codes: 200 OK (Here's the movie!), 404 Not Found (Sorry, we couldn't find that movie!), 500 Internal Server Error (Oops, something went wrong on our end!)

Create new movie

POST /movies

This endpoint is like adding a new movie to the library's collection. You need to provide all the movie details in a specific format, like filling out a form with title, description, genre, etc.

Request Body: 

                        {
                            "title": "string (required)",
                            "description": "string (required)",
                            "genre": "string (Genre Name - e.g., 'Action')",
                            "director": "string (Director Name - e.g., 'Quentin Tarantino')",
                            "actors": ["string (Actor Name)"],
                            "imagePath": "string (optional)",
                            "featured": "boolean (optional)"
                        }
                    

                    { "_id": "string (ObjectId)", "title": "string (required)", "description": "string (required)", "genre": { ... }, "director": { ... }, "actors": ["string (Actor Name)"], "imagePath": "string (optional)", "featured": "boolean", "__v": "number" }

Status Codes: 201 Created (The movie was successfully added!), 400 Bad Request (Oops, something was wrong with the information you provided!), 500 Internal Server Error (Oops, something went wrong on our end!)

Status Codes: 200 OK (The movie information was updated!), 404 Not Found (We couldn't find a movie with that ID!), 500 Internal Server Error (Oops, something went wrong on our end!)

Register new user

POST /users

This endpoint is like signing up for a library card so you can borrow movies. You need to provide your personal details like username, password, email, and birthday.

Request Body: 

                        {
                            "username": "string (required)",
                            "password": "string (required)",
                            "email": "string (required, valid email)",
                            "birthday": "date (ISO 8601, optional, e.g., 'YYYY-MM-DD')",
                            "firstname": "string (optional)",
                            "lastname": "string (optional)"
                        }
                    

                    { "_id": "string (ObjectId)", "username": "string (required)", "email": "string (required)", "birthday": "date (ISO 8601, optional, e.g., 'YYYY-MM-DD')", "favoriteMovies": ["string (list of movie ObjectIds - movies the user has marked as favorites)"], "__v": "number" }

Note: The password is never returned in responses for security reasons.

Status Codes: 201 Created (Welcome! Your account was created successfully!), 400 Bad Request (Oops, something was wrong with the information you provided!), 500 Internal Server Error (Oops, something went wrong on our end!)

Update user information

This endpoint is like updating your contact information on your library card, such as your address or phone number. You use your username to identify your account and then provide the updated details.

Request Body: (all fields are optional, but at least one must be provided) 

                        {
                            "password": "string (new password - will be hashed)",
                            "email": "string (valid email)",
                            "birthday": "date (ISO 8601, optional, e.g., 'YYYY-MM-DD')",
                            "firstname": "string (optional)",
                            "lastname": "string (optional)"
                        }
                    

                    { "_id": "string (ObjectId)", "username": "string", "email": "string", "birthday": "date (ISO 8601, optional, e.g., 'YYYY-MM-DD')", "favoriteMovies": ["string (list of movie ObjectIds - movies the user has marked as favorites)"], "__v": "number" }

Note: The password is never returned in responses for security reasons.

Status Codes: 200 OK (Your information was updated!), 404 Not Found (We couldn't find a user with that username!), 500 Internal Server Error (Oops, something went wrong on our end!)
